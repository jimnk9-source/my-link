# 마이링크 (MyLink) - 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 사용자가 여러 링크(SNS, 쇼핑몰, 블로그, 웹사이트 등)를 하나의 통합된 페이지에서 관리하고 타인에게 쉽게 공유할 수 있는 프로필 링크 서비스 제공 (Linktree 클론)
- **대상 사용자**:
  - 다양한 채널을 운영하는 크리에이터 및 인플루언서
  - 여러 비즈니스 링크와 연락처를 하나로 모아 홍보해야 하는 소상공인
  - 자신만의 정리된 외부 링크 페이지를 공유하고자 하는 일반 개인 사용자

## 2. 핵심 기능 목록 (MVP)

해당 프로젝트는 빠르고 명확한 목적 달성을 위해 아래의 핵심 필수 기능에 집중합니다. **(테마 커스텀 및 이미지 업로드 기능 제외)**

### 2.1 회원 관리
- **회원가입 및 로그인 (인증)**: 구글(Google) 계정 연동 또는 지메일 기반 가입을 통한 세션 관리
- **회원 탈퇴 정책**: 회원이 서비스를 더 이상 사용하지 않고 탈퇴하더라도 그동안 누적된 통계 및 기존 링크 데이터 테이블은 **영구 보존(삭제 안 함)**됩니다.

### 2.2 프로필 관리
- **이미지 없는 텍스트 프로필**: 별도 프로필 이미지 업로드 기능 없이 텍스트 기반의 프로필 제공
- **이름 및 식별자 체계 분리 (`displayName` & `username`)**:
  - **URL Slug / 고유 닉네임 (`displayName`)**: 퍼블릭 페이지 접속 시 URL 슬러그(`mydomain.com/[displayName]`)로 활용되는 핵심 식별자입니다. **가입 시 사용자의 구글 지메일(Gmail) 아이디 앞부분(예: `abc@gmail.com`의 경우 `abc`)을 추출하여 초기 값으로 자동 설정**됩니다.
  - **사용자 실명 (`username`)**: 페이지 화면에 추가로 노출되어 실제 사람의 이름이나 브랜드명(예: 홍길동, 지민규)을 직관적으로 보여주는 유저네임.
- **프로필 정보 수정 (인라인 편집 저장)**: 웹상에서 텍스트 영역을 클릭해 수정한 뒤 **키보드 엔터(Enter) 키를 누르거나, 마우스로 화면의 다른 아무 곳이나 빈 영역을 빈 곳을 클릭(포커스 아웃)**하면 즉시 변경 사항이 저장되는 빠르고 직관적인 **인라인 편집(Inline Edit)** 기능 지원.

### 2.3 링크 관리 (CRUD 및 제어)
- **새로운 링크 추가**: 링크 리스트에 표시될 제목과 목적지(Target URL) 추가
- **링크 조회/수정/삭제**:
  - 등록된 링트 리스트 조회 및 개별 링크 영구 삭제 아이콘 제공
  - **인라인 편집(Inline Edit)**: 기존에 등록된 링크의 제목이나 URL 텍스트를 클릭하여 제자리에서 수정하고, 위의 엔터/포커스 아웃 규칙에 따라 자동 저장.
- **파비콘(Favicon) 구글 API 연동**: 사용자가 링크 URL 입력 시 각 링크 항목별로 구글 파비콘 API(`https://www.google.com/s2/favicons?domain=...`)를 호출해 해당 사이트의 로고(파비콘) 이미지를 자동 추출하여 표시
- **링크 활성화 여부 토글 (On/Off)**: 특정 링크를 삭제하지 않고도 상태값만 변경하여 접속자들에게 임시로 노출되지 않도록 숨기는 기능 
- **링크 순서 변경 (Drag & Drop)**: 목록 내에서 항목 조작 핸들 등을 통해 상하로 드래그 앤 드롭하여 직관적으로 최종 노출 순서 변경
- **통계 카운트 어뷰징 방지 (1분 제한)**: 링크 누적 클릭 통계 정확성을 위해 1명의 방문자(식별 가능한 트래커/IP 기준)가 아무리 연속으로 클릭하더라도 **1분에 1개까지만 카운트**되도록 철저히 제한.

### 2.4 퍼블릭 페이지
- **마이링크 외부 페이지**: `mydomain.com/[displayName]` 형태로 누구나 로그인 없이 접속해 정보를 확인하고 링크를 클릭할 수 있는 페이지 구성.
- **정보 노출 방식**: 상단 프로필 영역에 URL 슬러그로 사용된 고유 닉네임(`displayName`), 실제 사용자 실명(`username`), 그리고 간략한 소개글 텍스트 깔끔하게 노출. 
- **예외 처리 (404 마케팅)**: 존재하지 않는 URL로 접속할 시 '(404) 페이지를 찾을 수 없음' 에러 화면과 함께 신규 회원을 매력적으로 확보하기 위해 '나만의 마이링크 만들기' CTA(액션 유도) 버튼을 전략적으로 배치.
- **단일 테마 및 모바일 대응**: 사용자 정의 테마 기능 없이 통일된 단일 기본 뷰 제공. 관리자 화면 내 모바일 뷰 미리보기(Preview)는 별도로 제공하지 않으며 사용자는 브라우저 크기 조절을 통해 반응형 렌더링 확인 가능.

## 3. 기능 상세 및 주요 데이터베이스 요구사항

| 구분 | 기능명 | 상세 설명 |
| :--- | :--- | :--- |
| **회원 관리** | **인증 및 보존 데이터** | - 지메일(Gmail) 인증 시 이메일에서 `@` 앞부분(아이디)을 파싱하는 로직 포함<br>- 회원 탈퇴 액션 시 데이터 완전 삭제(Hard Delete)를 하지 않도록 정책 수립 |
| **프로필 관리** | **인라인 편집/이름 모델링** | - 저장소 제출(Form Submit) 방식 없이 `onBlur` 및 `onKeyDown(Enter)` 이벤트로 백단 API를 호출하여 Auto-save 처리<br>- URL 슬러그인 **`displayName`**의 충돌 및 Unique Validation 점검 로직 포함 |
| **링크 관리** | **동작 제어/트래핑** | - 썸네일/아이콘 이미지 업로드 비생성 (구글 파비콘 대체 대체)<br>- 활성/비활성 제어를 위한 `isActive`(Boolean)<br>- 순서 `order` 인덱스 값 핸들링<br>- DB/캐싱(Redis) 기반으로 각 IP 혹은 방문자 식별자 별 클릭 Timestamp를 기록하여 1분 이내 중복 Request 무시 체계 수립 |
| **퍼블릭 페이지** | **반응형 환경 구축** | - `[displayName]`를 키값으로 쿼리를 보내 유저 정보 데이터 Fetch<br>- 테마와 관련된 색상 코드 등의 스타일 DB 모델링 비존재 |
