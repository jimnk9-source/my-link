# 마이링크 (MyLink) - 사용자 시나리오 (User Scenario)

## 1. 소유자 (Owner) 시나리오: 계정 및 링크 관리 과정

### 1-1. 회원가입 및 프로필 관리
- 사용자는 퍼블릭 프로필 서비스인 마이링크를 시작하기 위해 구글(Gmail) 계정 연동 버튼을 클릭하여 회원가입과 로그인을 동시에 진행한다.
- 사용자는 방문자들에게 자신의 실제 이름 또는 브랜드명을 명확하게 노출하기 위해 프로필 상단의 초기화된 실명 영역(`username`)을 클릭하여 텍스트를 수정한 뒤, 키보드의 엔터(Enter)를 치거나 화면의 다른 영역을 마우스로 클릭하여(포커스 해제) 변경된 이름을 즉시 저장한다.
- 사용자는 자신만의 기억하기 쉬운 고유한 접속 주소를 갖기 위해 프로필 내 URL 슬러그 영역(`displayName`)을 클릭하여 수정한 뒤 키보드 엔터나 빈 곳을 클릭하여 저장한다.
- 사용자는 다른 소유자가 이미 점유한 주소를 뺏거나 겹치는 것을 방지하기 위해 `displayName` 수정 시 표시되는 실시간 중복 검사 메시지를 체크하며 닉네임을 사용 가능한 이름으로 변경 후 자동 확정한다.
- 사용자는 페이지에 접속한 방문자에게 자신을 간략히 어필하기 위해 프로필 하단의 소개글(Bio) 텍스트 영역을 클릭하여 내용을 기입하고, 마우스를 바깥 영역에 클릭함으로써 작성된 단어들을 손쉽게 실시간 서버에 저장한다.

### 1-2. 링크 관리 (추가, 수정, 삭제)
- 사용자는 자신의 다양한 외부 웹사이트나 SNS 채널을 방문자에게 공유하기 위해 '+ 링크 추가' 버튼을 눌러 빈 링크 항목 폼을 생성한다.
- 사용자는 기존에 등록된 링크의 제목이나 실제 목적지 주소(URL)를 변경하기 위해 해당 텍스트 부분을 클릭하여 페이지 이탈 없이 텍스트를 기입하고 수정 완료를 위해 엔터를 쳐 수정값을 반영한다.
- 사용자는 한 번 등록한 주소가 시각적으로 쉽게 인지되도록 지원받기 위해 URL 입력 후 저장 완료 시 자동 구동되는 구글 파비콘 API를 통해 연동된 사이트 아이콘 이미지를 확인한다.
- 사용자는 더 이상 운영하지 않거나 잘못 입력된 유효하지 않은 항목을 정리하기 위해 개별 링크 카드에 위치한 '삭제' 기능 아이콘을 클릭하여 리스트에서 안전하게 제외시킨다. (단, 소유자가 계정을 탈퇴해도 이전 정보들은 영구히 유지된다.)
- 사용자는 특정 링크 항목을 목록에서 지우지 않고 단기 이벤트 종료 등의 사유로 일시적으로만 방문자에게 숨기기 위해 해당 링크의 상태를 변경하는 활성화/비활성화 '토글(Toggle)' 스위치를 조작한다.
- 사용자는 방문자가 가장 먼저 누르기 바라는 최우선순위 링크를 맨 위에 배치시키기 위해 링크 리스트에서 항목별 조작 핸들을 잡아 위아래로 끌어다 놓는 드래그 앤 드롭 방식을 채택한다.

### 1-3. 데이터 통계 조회
- 사용자는 자신이 등록해둔 여러 개의 목적지들 중 어떤 쪽으로 트래픽이 집중되는지 파악하기 위해 관리 화면에 노출된 각 '링크 누적 클릭 수(Click Count)' 숫자를 확인한다. (이때, 신뢰성을 확보하기 위해 동일한 방문자가 누른 것은 시간당 1분에 한 건씩만 횟수에 등록되어 있는 정확한 수치를 확인하게 된다.)

## 2. 방문자 (Visitor) 시나리오: 페이지 방문 및 전환 과정

### 2-1. 접속 및 탐색
- 사용자는 특정 소유자가 운영하는 채널 리스트와 기본 프로필 정보를 확인하기 위해 소유자가 SNS 프로필 등에 남겨둔 고유 URL(`mydomain.com/[displayName]`)을 입력/클릭해 마이링크 방문자 뷰에 입장한다.
- 사용자는 모바일 기기에서도 버튼이나 글씨가 잘리지 않고 편안하게 탐색하기 위해 디바이스 특성에 맞추어진 좁은 화면 최적화 반응형 뷰어를 통해 내용을 스크롤하며 확인한다.
- 사용자는 소유자가 유도한 타겟 목적지(예로우: 쇼핑몰 메인 화면, 인스타그램 프로필 등)로 방문을 확장하기 위해 뷰어 화면상에 수직으로 나열된 특정 링크 항목 버튼을 직접 클릭한다. (클릭 시 소유자의 클릭 통계가 오르며, 방문자가 무분별하게 버튼을 연타하여도 서버 제어 시스템에 따라 1인당(1명당) 1분에 1번만 합산 처리된다.)

### 2-2. 인지 및 예외 상황 
- 사용자는 자신이 클릭한 개인 링크주소가 정상적인 URL 규격이 아니어서 도달할 수 없을 때 발생할 불편함을 최소화하기 위해 접속 시 브라우저가 기본 안내하는 예외 경고 메시지를 확인한다.
- 사용자는 소유자가 설정을 변경해 더 이상 존재하지 않거나 닉네임이 바뀐 마이링크 주소에 뒤늦게 접속한 상황을 인지하기 위해 '(404) 페이지 존재하지 않음' 예외 안내 화면을 마주한다. 사용자는 에러 문구 하단에 매력적으로 위치한 "나만의 마이링크 만들기" 버튼(CTA)을 호기심에 클릭해 마이링크 가입 프로모션(퍼널) 단계로 자연스럽게 유입된다.
